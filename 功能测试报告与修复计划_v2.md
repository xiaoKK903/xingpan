## 三、发现的问题

### 问题 1：星盘名称中文显示为乱码 ⚠️ 中等

**现象**：`/my-charts` 页面中，已保存的星盘名称显示为 "????" 而非正确的中文。

**原因分析**：
数据库中的 `name` 字段存储了中文，但读取后显示为乱码。可能的原因：

1. **数据库字符集问题**：MySQL `CHARACTER SET` 可能不是 `utf8mb4`，导致中文无法正确存储/读取
2. **API 响应编码问题**：后端返回 JSON 时可能未正确设置 UTF-8 编码
3. **前端显示问题**：Vue 组件在渲染时未正确处理中文字符

**验证方法**：
```sql
-- 检查数据库字符集
SHOW FULL CREATE TABLE charts\G
-- 或检查单行数据
SELECT id, name, HEX(name) FROM charts WHERE name LIKE '%测试%';
```

**修复方案**：
1. 检查并修改数据库表的字符集：`ALTER TABLE charts MODIFY name VARCHAR(200) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;`
2. 检查 MySQL 连接字符串是否指定了 `charset=utf8mb4`

---

### 问题 2：每日星运 API 参数格式 ⚠️ 已排除

**之前认为有问题**：前端传递英文星座名 "Aries" 导致后端 400 错误

**实际情况**：经代码核查，前端从 `/horoscope/signs` API 获取星座列表（返回中文名称如 "白羊座"），然后将中文名称作为参数调用 `/horoscope/today?sign=白羊座`。**没有问题**。

**现象**：VIP 状态接口返回 `is_vip: false`，这是正常的——测试用户不是 VIP 会员。

### 问题 4：话题挑战显示"暂无活跃话题" ℹ️ 数据问题
